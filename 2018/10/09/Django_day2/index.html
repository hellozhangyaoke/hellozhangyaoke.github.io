    Django_day2(快速回忆项目架构)

    • Templates:下面放的是前段页面
      因为django会根据setting里app的顺序区寻找Template，所以建立了Template目录后还要在目录下建立当前app名字的目录，把前端文件放在app名字的目录下：

    • include的作用：
      首先要在app路径下建立urls.py，

      • localhost:8000/blog/1

        • 根目录urls.py:
          from django.contrib import admin
from django.urls import path,include
urlpatterns = [
path('admin/', admin.site.urls), 
path('polls/',include('polls.urls')), #polls是主路径，后面跟着的是polls.urls.py里面的路径
path('blog/',include('blog.urls'))
]
        • polls的urls.py
          from django.urls import path
from . import views

        urlpatterns = [
        path(‘1/‘,views.index), #此时访问的路径为 ‘polls/1/’
        path(‘2/‘,views.index1), #此时访问路径为 ‘polls/2’
        ]

    • Models
      先在app路径下建立migrations_文件夹
      _app下models.py:

      from django.db import models

    Create your models here.

    class Article(models.Model): #数据表名称，在数据库里显示的是’polls.Article’
    title = models.CharField(max_length=32,default=’Title’)
    content = models.TextField(null=True)

    
**生成后要执行数据迁移：**

`python3 manage.py makemigrations `
`python3 manage.py migrate`

**这时migrations路径下已经有有了文件：**

**查看_0001_initial.py_，数据表的信息:**

    from django.db import migrations, models

    class Migration(migrations.Migration):

    initial = True

dependencies = [
]

operations = [
    migrations.CreateModel(
        name='Article',
        fields=[
            ('id', models.AutoField(auto_created=True, primary_key=True, serialize=False, verbose_name='ID')),
            ('title', models.CharField(default='Title', max_length=32)),
            ('content', models.TextField(null=True)),
        ],
    ),
]
